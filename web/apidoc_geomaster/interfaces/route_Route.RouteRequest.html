Optional wayPoint

wayPoint?: string | IPoint | IPoint[]

경유지 정보 다음 중 하나의 위치 정보 형태를 지원한다. 지리적 위치 정보 (위도,경도 - WSG 84 좌표) 좌표에 prefix 'v'를 붙여 via(pass through)로 도착 시킬 수 있음. 여러 개의 좌표를 지원한다. ("+"로 구분) 좌표가 여러 개일 경우, 가장 최적인 좌표로 도착한다.

Example

v961230,1945823 + 960891,1945933 + 960945,1946124
링크ID
해당 도로를 지나가도록 경로 탐색
여러 개의 링크ID를 지원한다. ("+"로 구분)

Example

5627200048983599 + 5627200049015272
해안도로ID
해안도로를 지나가도록 경로 탐색.
여러 개의 도로ID를 지원하지 않음.
연결링크
해당 도로연결을 지나가도록 경로 탐색
도로 연결은 구분자 "_"(underscore)로 연결을 표현한다.
여러 개의 도로연결을 지원한다. ("+"로 구분)

Example

5627200049003087 _ 5627200048983599
경유지는 여러 개 설정이 가능하며, 갯수 제한은 최대 12개로 제한한다.
각 경유지는 구분자 ";"(semicolon)으로 구분한다.

Example

5627200048983599 + 5627200049015272 ; 954908,1951636 + 954907,1951836 ; 47 ; 5627200049003087 _ 5627200048983599
경유지1: 5627200048983599링크의 시작위치 또는 5627200049015272링크의 시작위치. 두 링크 중 더 최적 경로로 지나감.
경유지2: 좌표1(954908,1951636), 좌표2(954907,1951836) 중에 하나. 더 최적 경로로 지나감.
경유지3: 해안도로47번길
경유지4: 5627200049003087링크를 지나 5627200048983599 링크의 시작위치.
